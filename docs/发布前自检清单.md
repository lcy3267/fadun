# 发布前自检清单

在合并分支、打 tag、或部署到生产环境前，可按本清单快速扫一遍。未全部通过也可上线，但应清楚风险点。

---

## 1. 构建与依赖

- [ ] 根目录执行 `npm run build` 能通过（含 client workspace）。
- [ ] 生产机/CI 安装依赖策略与本地一致（例如需 `vite` 等 dev 依赖参与 build 时，勿用会剔除 devDependencies 的安装方式）。
- [ ] `package-lock.json` 已提交且与 `package.json` 一致（避免环境装到不同版本）。

## 2. 环境变量与密钥

- [ ] 服务器存在正确 `.env`（或等价注入），**勿**把真实 Key 提交进仓库。
- [ ] `JWT_SECRET` 在生产环境为足够长的随机串。
- [ ] `DATABASE_URL` 指向预期 SQLite 路径（或未来迁移后的库）。
- [ ] LLM：`LLM_PROVIDER`、`OPENAI_*` / `ANTHROPIC_*` 等与当前厂商一致。
- [ ] RAG / 聊天：`EMBEDDING_*` 与 Chat 可分离配置；换过 embedding 模型后需重建案件索引。
- [ ] 前后端分域：`VITE_API_BASE`、`CORS_ORIGIN` 已包含线上页面 origin。
- [ ] 若启用公开检索：`SERPAPI_API_KEY` 有效且额度充足。

## 3. 安全与权限

- [ ] 证据下载、删除、上传等接口均走鉴权，且仅能访问**本人案件**数据。
- [ ] 证据文件通过 `/api/evidence/uploads/:id` 等受控路由读取，避免静态目录暴露上传文件。
- [ ] 上传类型与白名单（MIME/扩展名）仍符合预期。

## 4. 核心业务路径（建议手测）

- [ ] 注册 / 登录 / 拉取案件列表正常。
- [ ] 上传证据（图片 + 至少一种文档：pdf/docx/xlsx/txt）→ 解析任务完成。
- [ ] 批量「证据归类认证」任务完成，列表分组与草稿箱符合预期。
- [ ] 案件综述（若使用）能生成且无裸 Key 暴露在响应里。
- [ ] 案件聊天：流式输出正常；SSE / 代理超时在生产环境足够长。
- [ ] RAG：`server/.rag/evidence_case/<caseId>/` 在认证成功后有更新预期（或接受首次为空后再跑索引任务）。

## 5. 前端与静态资源

- [ ] 线上打开首页与主要路由无 404（SPA fallback / Nginx 配置正确）。
- [ ] 图片证据在登录态下可预览（同源代理或 `VITE_API_BASE` 正确）。
- [ ] 移动端关键页无明显横向撑破（证据标题、弹窗等）。

## 6. 仓库与产物

- [ ] 未误提交 `.env`、本机 `*.db`（若策略为不提交数据库）。
- [ ] 未提交本机 RAG 索引目录 `server/.rag/`（应由任务在环境内生成）。
- [ ] 未提交 `.DS_Store`、大体积日志。

## 7. 部署与进程（若用 Node 直跑 / PM2 等）

- [ ] 监听端口与防火墙 / 安全组一致。
- [ ] 反向代理将 `/api` 指到后端，且 SSE 关闭缓冲（`proxy_buffering off` 等）。
- [ ] 重启后上传目录 `server/uploads/` 持久化路径未丢。

---

**说明**：清单可按团队习惯增删；与具体版本功能不一致时，以 README 与 `.env.example` 为准。
